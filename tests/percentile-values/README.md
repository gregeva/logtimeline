# Percentile-Value Regression Harness

## Purpose

Detect regressions in ltl's percentile calculations against a set of golden log
files. Complements two adjacent test suites:

| Suite | What it checks | When it breaks |
|---|---|---|
| `tests/validate-regression.sh` | Rendered terminal output is byte-identical (layout + column rendering) | A layout or formatting change |
| `tests/validate-percentile-mode.sh` | The `-V` percentile-mode block emits the locked contract surface | A change to the `-V` audit contract |
| `tests/validate-percentile-values.sh` (this one) | Numeric percentile values in `-o` CSV output drift within tolerance from a captured baseline | The arithmetic that produces percentile values changes |

Unlike `validate-regression.sh` (which is byte-identical), this harness
permits non-byte-identical drift and classifies it into tiers.

## Scope

- Compares two CSV outputs per scenario: `*-LTL-MESSAGES-*.csv` and
  `*-LTL-STATS-*.csv`, both produced by `ltl -o`.
- 12 scenarios = 4 golden log files × 3 canonical option sets.
- 24 baseline CSV files total (`baselines/<scenario>/{messages,stats}.csv`).
- No `ltl` code changes are part of this harness. It is test infrastructure.

## Tier Model

Every per-cell comparison is classified into the worst tier breached:

| Tier | Name | Rule | Blocking? |
|---|---|---|---|
| **T1** | Exact | Byte-identical | No (advisory) |
| **T2** | Tight | `abs(new − old) ≤ 1% × old` | No (advisory) |
| **T3** | Loose | `abs(new − old) ≤ 5% × old` | **Yes** |
| **T4** | Structural | Same key set; per-row monotonicity `P50 ≤ P90 ≤ P99 ≤ P99.9`; same row count | **Yes** |

T1 and T2 are reported but do not fail the harness. T3 and T4 fail the harness
(exit 1). All failures — including advisory T1/T2 entries when `--show-all` is
given — print the full context: scenario, which file (MESSAGES or STATS),
which key, which column, baseline value, new value, deviation, and the rule
that was breached.

## Usage

```bash
# Run all 12 scenarios. Reports T3/T4 failures; exits 1 if any.
./tests/validate-percentile-values.sh

# Show T1/T2 advisory drift in addition to T3/T4 failures.
./tests/validate-percentile-values.sh --show-all

# Run a single scenario.
./tests/validate-percentile-values.sh --scenario codebeamer-daily

# Regenerate all baselines from the current ltl. Requires explicit
# confirmation since baselines are deliverables.
./tests/validate-percentile-values.sh --regenerate

# Regenerate a single scenario's baseline.
./tests/validate-percentile-values.sh --regenerate --scenario codebeamer-daily
```

## Adding a Scenario

1. Append a row to `scenarios.tsv` (tab-separated: `name`, `file`, `options`).
2. Run `./tests/validate-percentile-values.sh --regenerate --scenario <name>`
   to capture the baseline.
3. Verify the baseline by running the harness without `--regenerate`; the
   new scenario should pass at T1 (byte-identical to what was just captured).
4. Commit `scenarios.tsv` and `baselines/<name>/{messages,stats}.csv` together.

## Regenerating Baselines

Baselines are **deliverables**, not disposable artifacts. Do not regenerate
casually:

- Regenerate only when the new values are known-correct (e.g. after an
  intentional change to percentile computation that has been reviewed).
- The `--regenerate` flag prompts for confirmation before overwriting.
- Always inspect the diff of the regenerated baselines before committing.
- When regenerating, also update the `Generated by ltl version` field in
  each baseline's header comment (the harness writes this automatically).

## What Drifts Trigger Which Tier (typical examples)

- **T1 only (byte-identical)** — no change to percentile arithmetic.
- **T2 (within 1%)** — small interpolation or rounding tweaks; usually
  intentional.
- **T3 (within 5%)** — algorithmic change (e.g. different percentile
  interpolation method). Block until reviewed.
- **T4 (structural)** — key set changed, ordering of percentiles violates
  monotonicity, or row count differs. Usually a bug or a major refactor.

## Output Format

Per failure (one line each):

```
FAIL [T3] scenario=codebeamer-daily file=messages key="plain|[200] GET /…" column=p99 baseline=1234 new=1320 deviation=6.97% rule="abs(new-old) > 5% * old"
```

Per scenario summary:

```
codebeamer-daily: 248 cells checked, 0 T4, 0 T3, 2 T2, 5 T1, structural=OK
```

Exit codes:

- `0` — no T3/T4 failures (T1/T2 advisories may have been printed)
- `1` — at least one T3 or T4 failure
