# Release v0.15.1

## Bug Fixes

- `--help`, `--explain`, and `--version` now take precedence over option parsing and validation — they print their output and exit before any file is read, even alongside a malformed option; when combined, `--help` wins over `--explain` over `--version`. Unique-prefix abbreviations of long options are no longer accepted, and invalid command-line options now produce a specific message naming the option instead of a generic error. (#309)
- Invalid command-line options and other command-line errors are now shown as a single block on standard error — the parser's message and ltl's error line appear together, separated from the usage hint by a blank line, with the error emphasized in bold rather than red. The internal `Died at ...` trace no longer leaks. (#308)
