# Release v0.14.5

## What's New

- Re-running ltl against files it has already analyzed is now faster: prior runs' bound information is read from `ltl-index.csv` at startup, skipping the initial bound-discovery scan (#179)
- Histogram bottom axis (`-hg`) now shows tick marks at each percentile position so legend values line up visually with the distribution (#185)
- New `--percentile-precision` / `-pp` (tier 1–9) and `--percentile-buckets-per-decade` / `-pbpd` flags to tune percentile accuracy; `--exact-percentiles` / `-ep` opt-out for the previous sort-based computation (#189)
- Heatmap (`-hm`) and histogram (`-hg`) use significantly less memory on larger files (see Benchmark Comparison below). Output is visually identical to previous releases. Use `--exact-percentiles` to fall back to the previous behavior (#34)

## Bug Fixes

- (none)

## Benchmark Comparison (v0.14.4 → v0.14.5)

The bin-counter migration's payoff shows up on heatmap (`-hm`) and histogram (`-hg`) runs against larger files. Timing is essentially flat (within run-to-run noise); the wins are in peak memory.

### Heatmap and Histogram Tests — Memory

| Test | v0.14.4 | v0.14.5 | Delta |
|------|---------|---------|-------|
| single-day-access-log (762K lines) — `-hm` | 159.8 MB | 120.1 MB | **-25%** |
| single-day-access-log — `-hg` | 286.3 MB | 154.6 MB | **-46%** |
| single-day-access-log — `-hm -hg` | 292.5 MB | 120.8 MB | **-59%** |
| single-day-access-log — `-hm -hg -g` | 331.3 MB | 167.1 MB | **-50%** |
| multi-day-custom-logs (1.5M lines) — `-hm` | 206.9 MB | 179.6 MB | **-13%** |
| multi-day-custom-logs — `-hg` | 243.0 MB | 199.2 MB | **-18%** |
| multi-day-custom-logs — `-hm -hg` | 249.7 MB | 177.2 MB | **-29%** |
| multi-day-custom-logs — `-hm -hg -g` | 289.4 MB | 232.4 MB | **-20%** |

### Heatmap and Histogram Tests — Timing

| Test | v0.14.4 | v0.14.5 | Delta |
|------|---------|---------|-------|
| single-day-access-log — `-hg` | 11.4 s | 11.3 s | **-1%** |
| single-day-access-log — `-hm -hg` | 12.0 s | 11.8 s | **-2%** |
| multi-day-custom-logs — `-hg` | 17.0 s | 16.7 s | **-2%** |
| multi-day-custom-logs — `-hm -hg` | 17.3 s | 17.1 s | **-1%** |

### Known Regressions

**`-hm -hg -g` on single-day-access-log:** runtime up 12% (14.4 s → 16.2 s) despite memory dropping 50%. Combining all three heavy-feature flags has a small interaction cost worth a follow-up profile. Other `-g` combinations regressed by ≤9% or improved.

**Small-file heatmap/histogram memory:** files under ~100 MB show a 2–7% memory increase (~2 MB absolute). The new histogram backend has a small fixed overhead that's proportionally visible when the file is too small for the savings to dominate. On any file larger than that, the migration is a clear net win — see table above.
