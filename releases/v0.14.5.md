# Release v0.14.5

## What's New

- Re-running ltl against files it has already analyzed is now faster: prior runs' bound information is read from `ltl-index.csv` at startup, skipping the initial bound-discovery scan (#179)
- Histogram bottom axis (`-hg`) now shows tick marks at each percentile position so legend values line up visually with the distribution (#185)
- New `--percentile-precision` / `-pp` (tier 1–9) and `--percentile-buckets-per-decade` / `-pbpd` flags to tune percentile accuracy; `--exact-percentiles` / `-ep` opt-out for the previous sort-based computation (#189)
- Heatmap (`-hm`) and histogram (`-hg`) use significantly less memory on larger files (see Benchmark Comparison below). Output is visually identical to previous releases. Use `--exact-percentiles` to fall back to the previous behavior (#34)

## Benchmark Comparison (v0.14.4 → v0.14.5)

The bin-counter migration's payoff shows up on heatmap (`-hm`) and histogram (`-hg`) runs across all file sizes, with the largest absolute memory wins on extra-large datasets. Timing is essentially flat on standard non-consolidation scenarios and improves on std-tier consolidation (`-g`); the consolidation phase on extra-large files regresses substantially — see Known Regressions below.

### Known Regressions

**Consolidation phase (`-g`) on extra-large files:** runtime on month-scale access-log files regresses substantially when consolidation is enabled, with the worst impact on the largest test. The regression is concentrated in the `group_similar` phase and scales sharply with dataset size. Std-tier consolidation scenarios (files up to ~500 MB) all improve in v0.14.5 (-2% to -22%); the regression appears only at XL scale.

| Test | v0.14.4 | v0.14.5 | Delta |
|------|---------|---------|-------|
| month-single-server-access-logs (~12M lines, 1.5 GB) — `-n 25 -g` | 2.4 min | 4.3 min | **+82%** |
| month-single-server — `-hm -hg -g` | 3.0 min | 4.2 min | **+41%** |
| month-many-servers-access-logs (~39M lines, 7.6 GB) — `-n 25 -g` | 10.9 min | 30.4 min | **+179%** |
| month-many-servers — `-hm -hg -g` | 14.3 min | 28.4 min | **+99%** |

This regression has been observed and quantified through full v0.14.4 → v0.14.5 benchmarking (49 tests, std and XL); the root cause has not been investigated yet and requires follow-up. Reference data point: `group_similar` on month-many-servers `-n 25 -g` measured 24.0 s in v0.14.4 and 1102.4 s in v0.14.5. The full v0.14.2 → v0.14.3 → v0.14.4 → v0.14.5 trajectory at XL for this metric is captured in the issue #142 comment thread.

**Small-file heatmap/histogram memory:** files under ~100 MB show a 2–7% memory increase (~2 MB absolute). The new histogram backend has a small fixed overhead that's proportionally visible when the file is too small for the savings to dominate. On any file larger than that, the migration is a clear net win — see table above.
