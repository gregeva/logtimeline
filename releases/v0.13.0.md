# Release v0.13.0

## What's New

- Fuzzy message consolidation (`-g`) automatically discovers and merges similar log messages, reducing noise and revealing patterns that would otherwise be buried across thousands of unique entries. Checkpoint-based processing keeps memory low even on multi-gigabyte files — 88% memory reduction at production scale (#96)

### Performance — 7.9 GB, 150 access log files, 40.6M lines, 6.6M unique keys

| Mode | Lines | Unique keys | Patterns | S1 absorbed | Time | Memory |
|------|-------|-------------|----------|-------------|------|--------|
| Baseline (no `-g`) | 40.6M | 6.6M | — | — | 390s | 3,661 MiB |
| `-g 80 --no-final-pass` | 40.6M | 6.6M | 393 | 99.9% | 473s | 437 MiB |
| `-g 80` | 40.6M | 6.6M | 393 | 99.9% | 489s | 451 MiB |

393 discovered patterns absorbed 99.9% of 6.6M unique keys inline during parsing, preventing them from consuming memory. Time overhead decreases with scale (1.32× on small files → 1.21× at production scale). Memory crossover occurs between 1-5 files — at production scale, consolidation saves 3.2 GB of RAM.

## Bug Fixes

- `-g` as last option on command line no longer errors (#96)
