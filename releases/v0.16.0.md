# Release v0.16.0

## What's New

- User-defined metrics gain counting aggregations — `count`, `distinct`, `ratio`, `rate`, `drate` — with full string-token support, plus a token-key spec field that separates the extraction token from the column label; the canonical UDM average keyword is now `mean` (`avg` still accepted), so duplicate-name collision headers and CSV columns read `x:mean` instead of `x:avg` (#313)
- Numeric criteria can now highlight instead of filter: `-hdmin`/`-hdmax`, `-hbmin`/`-hbmax`, `-hcmin`/`-hcmax` mark entries inside an inclusive duration/bytes/count band across every highlight surface while keeping the full population in view, composing with the hard filters and with `-h` (AND) (#312)

## Bug Fixes

- `-dmin`/`-dmax` boundaries are now inclusive, matching `-bmin`/`-bmax`/`-cmin`/`-cmax`: entries exactly at the threshold are kept where they were previously dropped (#312)
- The STATS CSV `occurrences` total is confirmed as all lines in the bucket (plain plus highlighted) and is now locked by a test invariant; a dead exclusion alternative in its accumulation regex that contradicted this was removed (#320)
- Numeric filters now report how many lines they excluded solely for carrying no value for the filtered metric, instead of silently shrinking the analyzed population; the metric-carrying requirement is documented in `--help` and the usage guide (#321)
- An inverted numeric range (minimum above maximum) on any of the twelve filter or highlight range options now warns up front that the range is unsatisfiable, instead of producing an empty result or highlighting nothing with no explanation (#322)
- MESSAGES CSV numeric user-defined-metric columns now honor `--csv-precision` like every other numeric column, instead of emitting raw stored floats with up to 13 decimals (#324)
- Counting-aggregation user-defined metrics no longer emit an uninitialized-value division warning per message during statistics calculation; the CSV harness now fails any scenario whose stderr carries Perl runtime warnings (#326)
- `-hm` and `-hg` metric names now resolve through one shared parser: `-hg` accepts user-defined metric names with their original case (previously lowercased and rejected as unknown), `-hm` accepts built-in names case-insensitively (`-hm DURATION`/`-hm TIME` previously fell through to filename handling), and both options' documentation lists UDM metric selection (#327)
- A CSV file swept into a multi-file input glob no longer crashes a `-udm` run with a fatal timestamp-parse error; CSV rows whose timestamp is neither epoch nor ISO are skipped with a warning naming the file, line, and value (#328)
- MESSAGES CSV rows for messages with no duration data now emit blank duration totals instead of `0`/`0 usec`, so a measured zero total is distinguishable from no data; the CSV harness family rules regroup `impact` with the duration columns and split `std_dev`/`cv` into a dispersion family matching their documented populate thresholds (#330)
- The CSV structural harness now fails hard when a produced CSV carries a column absent from the rules spec, matching the drift engine's all-or-nothing coverage so new output columns cannot ship unspecified (#335)
- Access logs in standard/common format (no request-duration field) no longer display latency statistics, fabricated 0ms percentiles, or the statistics variant of the messages table; format detection now classifies such logs as the common format, and mixed runs of files with and without durations no longer dilute percentiles with zero samples (#345)
- `-mem` now measures the message-statistics counter store, correcting an undercount of reported memory under `-mdm bin` (#346)
- Malformed duration values on corrupt access-log lines (concatenated records) are treated as absent instead of entering calculations and the on-disk index; every test harness now fails on Perl runtime warnings from ltl, and the regression fixtures moved off the corrupt sample log (#341)
