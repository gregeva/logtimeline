# Release v0.14.4

## Bug Fixes

- Fix `--terminal-width` not applied before log key truncation — CLI overrides were parsed after auto-detected defaults, so `$max_log_message_length` never reflected the `--terminal-width` value (#157)
- Fix log key truncated to terminal width breaking consolidation on long messages — when `-g` is active, use 350-char cap (same as CSV) instead of terminal width, preserving UUIDs and other variable content needed for similarity detection (#158)
- Split S4 verbose metric into "S4 Pairwise discovery" and "S4 Re-scan absorbed" for better consolidation observability; align design documentation with implementation (#162)
- Fix cross-cluster pattern merge testing subsumption in wrong direction — more general patterns now correctly absorb more specific ones regardless of match count ordering (#164)
- Add benchmark observability: hash entry counts (log_messages, log_occurrences, log_stats), terminal/bucket config to benchmark data block and `-V` output (#166)
- Adjust CV decimal places based on magnitude — values under 10 show 2 decimals, 10-99 show 1 decimal, 100+ show no decimals. Previously always used 2 decimal places, causing values like `131.52` to overflow and wrap the line (#152)
- Redesign final consolidation pass with sliding window architecture — replaces unbounded streaming simulation with two-pass bounded design. On 7.7M lines: `group_similar` 64.5 min → 4.5s, RSS 3.2 GB → 1.8 GB. Removes 2M line auto-disable (#150, #144)
- Rename `--final-ceiling` to `--group-ceiling` (`-gc`) — user-facing control for consolidation scope. Remove internal tuning options from help and documentation (#150)
- Add `MEMORY_FINAL` end-of-execution memory snapshot to `-V -mem` output (#150)

## Benchmark Notes

Memory regressions reported in the v0.14.3 → v0.14.4 benchmark comparison (#166) were caused by terminal environment differences between runs, not code changes. The v0.14.3 baseline was captured in a non-TTY context (width=120, height=24) while the v0.14.4 benchmark was captured in a wider/taller terminal. Wider terminal → longer log keys → more unique entries in `%log_messages`.

Benchmark script now passes `--terminal-width 200` and `-bs 60` (when not explicitly set) for consistent results across terminal environments. Added `--options "<ltl options>"` passthrough for injecting additional options into all benchmark runs (#167).
