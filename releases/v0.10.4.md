# Release v0.10.4

**Release Date:** 2026-02-06

## Summary

This release adds highlight support to histograms and fixes a crash that occurred when processing large datasets.

## New Features

### Histogram Highlight Support (#42)

Histograms now support the `-h` (highlight) option, displaying the distribution of highlighted records overlaid on the total distribution.

**Features:**
- **Stacked bars** - Highlighted portion renders at the bottom in bright highlight color, non-highlighted portion on top in base color
- **Dual legend lines** - When highlight data exists, two legend lines are displayed:
  - Population percentiles in base metric color
  - Highlighted percentiles with highlight background color (highlighter effect)
- **Smooth transitions** - Uses half-block characters (▀▄) for smooth color transitions at boundaries

**Usage:**
```bash
# Show histogram with POST requests highlighted
./ltl -hg -h "POST" logs/access.log

# Show only duration histogram with specific pattern highlighted
./ltl -hg time -h "Metrics" logs/access.log
```

### Adaptive Histogram Width

Histogram width now automatically adapts based on the number of histograms displayed:
- 1 histogram: 80% of terminal width
- 2 histograms: 90% of terminal width
- 3 histograms: 99% of terminal width

The explicit `-hgw` option overrides this adaptive behavior when specified.

## Bug Fixes

### Memory Tracking Crash (#50)

Fixed a crash (SIGTRAP) that occurred after processing large datasets (~8.5GB / 16M lines). The issue was in the memory tracking code within `get_memory_usage()` which failed when taking memory snapshots after file processing completed.

**Symptoms:**
```
zsh: trace trap  ltl -bs 240 *2026-01-2?.txt
```

**Root cause:** Memory measurement code triggered a trap signal on macOS when processing very large datasets.
