# Release v0.14.3

## What's New

- Default Dice similarity threshold for `-g` and `--final-threshold` changed from 80% to 85%
- Added "Consolidating similar messages..." progress status display during fuzzy consolidation stage
- Benchmark script (`run-benchmark.sh`) now merges results when run incrementally — running `full` then `xl` separately no longer overwrites previous results
- Standardized NYTProf profiling workflow with observability framework: `run-profile.sh`, `extract-profile.pl` (with declarative `--checks-file` cross-validation), profile-ready contract, pre/post-profiling checklists, and diagnostic pattern index (#138)

## Bug Fixes

- Adaptive per-key eviction for consolidation working set — replaces stall detection with EMA-based survival thresholds; XL benchmark: 3.4× faster, 47.7% less memory (#135)
- Redesigned final pass to re-scan all `%log_messages` keys through the full S1→checkpoint pipeline — old final pass bypassed the architecture entirely; access log reduction improved 27.5% → 52.3% (#137)
- Auto-disable final consolidation pass when input exceeds 2M lines to prevent runaway processing times (#144)
- Replace `Win32::Process::Info` (WMI) with direct `K32GetProcessMemoryInfo` kernel32 syscall for Windows memory tracking; disabled by default on Windows until validated — opt in with `-mem` (#143)
