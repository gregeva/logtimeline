# Release v0.14.6

## Breaking Changes

- CSV outputs (`-o`) now use consistent snake_case lowercase column names across MESSAGES and STATS, aligned with internal metric identifiers (`%log_stats` keys). MESSAGES headers `MinDuration`, `MeanDuration`, `MaxDuration`, `StdDev`, `P1`–`P99.9`, `CV`, `TotalDuration`, `TotalDurationNice`, `TotalBytes`, `TotalBytesNice`, `MeanBytes`, `Occurrences`, `Category`, `Message`, `Impact` become `min`, `mean`, `max`, `std_dev`, `p1`–`p999`, `cv`, `duration`, `duration_nice`, `bytes`, `bytes_nice`, `mean_bytes`, `occurrences`, `category`, `message`, `impact`. STATS headers `totalOccurrences`, `bytesNice`, `durationNice` become `occurrences`, `bytes_nice`, `duration_nice`. External CSV consumers must update column references. (#221)
- SRE-grade distribution-analysis columns added to `-o` CSV outputs in both MESSAGES and STATS: body percentile `p25`, precomputed `iqr`, high-volume tail `p9999`, and three shape statistics — `skewness`, `kurtosis`, and `bimodality_coef` (Sarle's bimodality coefficient — multimodal when > 5/9 ≈ 0.555). Distribution shape can now be characterized from the CSV alone, enabling load-shape mode characterization, mode-evolution detection, and per-API certainty bounding without re-running ltl. The unused `z_score` column is removed from STATS (it was computed but never displayed, consumed, or documented). External CSV consumers must update column references. See `docs/usage.md` "Distribution shape" subsection for interpretation and sample-size notes. (#222)
- All computed statistics are now stored at full precision internally; rounding moves to the CSV-emit boundary. Default CSV emission uses per-family decimal counts derived from the resolved duration unit (`-du us` → 6 decimals on time-valued columns; `-du ms` → integer; shape statistics fixed at 3 decimals). Default-mode values previously rounded at storage (`mean`, `count_mean`, `std_dev`, `cv`, `skewness`, `kurtosis`, `bimodality_coef`, `impact`, percentile ladder) may shift in the last decimal because they are now computed from precise inputs and rounded only once at emit. External CSV consumers that compared values for byte equality should expect drift in trailing decimals; numeric comparison within float tolerance is unchanged. The `duration` column in both MESSAGES and STATS CSV is now `float` rather than `int` (sub-ms duration units produce fractional ms sums). The bar-graph and message-table CV cells preserve today's adaptive width formatting at the render boundary. (#268)

## What's New

- `-V` now accepts an optional section argument so harnesses and humans can request specific diagnostic sections (`-V <name>`, `-V a,b`, `-V list`, `-V all`); section delimiters and names are stability-contracted (#226)
- New `tests/HARNESS-DESIGN.md` codifies the test-harness ↔ application observability contract with naming rules, delimiter contract, must-fail-on-missing-anchor rule, self-documenting assertions, and 10 trap patterns with wrong-vs-right examples (#226, #238)
- `-V` section `histogram` renamed to `histogram-array` to reflect the underlying architecture (exact-percentile in-memory array path) and parallel `histogram-bin-counters` (HDR-style bin-counters path); per-metric dimensions formatting is now shared between the two via a single helper so future format changes apply to both (#237)
- New `--dark-background` / `-dbg` flag forces dark heatmap gradients (inverse of `-lbg`); wins over `-lbg` regardless of CLI order. New `-V heatmap-palette` section exposes the resolved palette state (active metric, light/dark selection, source of selection, gradient arrays) and a new `tests/validate-heatmap-palette.sh` harness asserts palette correctness across `-lbg`/`-dbg` × `duration`/`bytes`/`count` plus both precedence orderings (#250)
- `-so` / `--sort-on` now accepts percentile latency (`p1, p5, p10, p25, p50, p75, p90, p95, p99, p999, p9999, p99999`), distribution-shape metrics from #222 (`iqr, skewness, kurtosis, bimodality_coef`), and the previously-missing `mean_bytes`. New percentiles `p5`, `p10`, and `p99999` are also added to MESSAGES + STATS CSV outputs and to `%histogram_stats` (#220)
- New `--help statistics` shows a grouped index of every statistic ltl emits; new `--explain <topic>` shows long-form per-topic documentation with interpretation tables and worked examples. 12 topics ship: `min`, `max`, `mean`, `std_dev`, `cv`, `iqr`, `percentiles`, `skewness`, `kurtosis`, `bimodality_coef`, `heatmap`, and `histogram` (the last two include example charts) (#261)
- New `tests/validate-csv-output.sh` harness validates structural integrity of `-o` CSV outputs (MESSAGES and STATS): column presence and ordering, row/header alignment, required-vs-conditional population, functional group consistency across metric families, data-type correctness (numeric vs human-readable), and per-column fixed-decimal rules; release-process step inserted before benchmarks (#223)
- New explicit data-model selectors `-dm`/`--data-model`, `-hgdm`/`--histogram-data-model`, `-hmdm`/`--heatmap-data-model`, `-mdm`/`--message-stats-data-model`, `-bdm`/`--bucket-stats-data-model` pin the raw-array or bin-counter reduction path per surface; `--exact-percentiles` is now deprecated (equivalent to `--data-model raw` on histogram/heatmap) (#266)
- New `--csv-precision` / `-cp` flag controls how stored numeric values are emitted to `-o` CSV: `default` (per-family decimals derived from `-du`, suitable for humans and most consumers), `full` (raw precise floats with no rounding, suitable for external tools and harness baselines that need to observe sub-display-precision drift), or an integer `N` (cap every numeric column at N decimal places). Trailing zeros are never emitted; the family decimal count is a maximum, not a fixed width. New `-V csv-output` section surfaces the resolved precision mode, duration unit source, per-family decimal counts, and the structural max-decimals ceiling so harnesses can validate against the run's actual emit contract. (#268)
- New `tests/validate-statistics.sh` four-layer statistics test harness: Layer 1 drift against committed baselines, Layer 2 intra-row arithmetic invariants, Layer 3 external-oracle (NumPy/SciPy) algorithmic correctness for percentiles/std_dev/cv/skewness/kurtosis/bimodality, Layer 4 cross-model agreement between raw-array and bin-counter data models. Shared CSV cache at `tests/.artifacts/csv/` reused by `validate-csv-output.sh` (#223) eliminates duplicate `ltl` invocations. Release-process step inserted between structural validation and benchmarks (#224)
- New `-V percentile-algorithm` section declares the resolved data model and the effective percentile algorithm for each of the four percentile-emitting surfaces (histogram, heatmap, message-stats, bucket-stats). Layer 3 of the statistics test harness consumes the section to pick its reference computation; the oracle now dispatches on `nearest_rank` vs `exponential_interpolation_within_bucket` instead of always running linear interpolation, which eliminates the algorithm-mismatch L3 false positives observed on `p999`/`p9999`/`p99999` during #224 Phase F (#280)

## Bug Fixes

- Test harnesses that grep for renamed/missing anchors no longer silently pass — `capture-regression.sh`, `validate-regression.sh`, and `baseline/compare-results.sh` now hard-fail on missing anchors with a diagnostic naming the expected anchor (#238)
- `validate-regression.sh` no longer aborts mid-run on a regression failure (`set -e` + `pipefail` + intentional non-zero from `diff`) (#238)
- Stray `\r…<spaces>\r` line-clear sequences no longer leak through `--disable-progress`, so section markers and other output now anchor cleanly at column 1 (#226)
- `tests/validate-index-read-back.sh` now runs in ~6 seconds (down from 5–10+ minutes) by starting each scenario from a prebuilt fixture index instead of regenerating the full index against a 290 MB log per scenario; harness auto-generates the fixture on first run via `tests/fixtures/regenerate-index-readback-fixtures.sh` (#240)
- `tests/validate-index-read-back.sh` assertions now self-document — each declares the application invariant being tested, where in `ltl` it is produced, and the stability contract that makes it stable; failure output surfaces all three so failures don't require external archaeology. New `assert_command` helper added for assertions that aren't simple line greps (#239)
- STATS CSV rows now match the header column count; previously `Text::CSV` emitted short rows when trailing percentile/shape cells were `undef`, misaligning downstream consumers (#223)
- Rate columns (`err-rate_*`, `msg-rate_*`) in STATS CSV now format to 1 decimal place; previously emitted raw division output (12+ decimal places) (#223)
- STATS CSV latency stats (`min`, `mean`, `max`, `std_dev`, `p1`–`p99999`, `cv`, `skewness`, `kurtosis`, `bimodality_coef`) are now emitted independently of terminal width; previously the CSV `push` was nested inside the latency display branch and was silently skipped when the column was auto-hidden on narrow terminals (#263)
- `mean` is no longer integer-truncated before being read back into the variance, skewness, kurtosis, and bimodality_coef formulas in `calculate_statistics()`; the fractional part of the mean is now preserved through the math, eliminating systematic inflation of `std_dev` and `cv` and distortion of the shape statistics (#271)
- `ltl` output is now deterministic across back-to-back runs on the same input. Nine `sort`/`keys %hash` sites that previously relied on randomized Perl hash iteration order — including top-N message ranking, message- and threadpool-table display sorts, threadpool column ordering, discriminative trigram pick in `-g` consolidation, CSV separator auto-detection, and `-mem` HWM display — now apply a deterministic tiebreaker on the hash key, so MESSAGES and STATS CSV outputs are byte-identical between runs (#269)
- `tests/validate-regression.sh`, `tests/validate-help-layout.sh`, and `tests/validate-histogram-ticks.sh` now emit failures in the self-documenting multi-line form (asserts / produced_by / contract) per `tests/HARNESS-DESIGN.md`, so a failure read cold tells the reader which invariant broke and where to look. The `validate-runtime-config.sh` `--exact-percentiles` deprecation-warning anchor was also corrected after the warning text was rewritten in #266 (#275)
