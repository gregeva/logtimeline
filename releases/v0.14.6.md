# Release v0.14.6

## What's New

- `-V` now accepts an optional section argument so harnesses and humans can request specific diagnostic sections (`-V <name>`, `-V a,b`, `-V list`, `-V all`); section delimiters and names are stability-contracted (#226)
- New `tests/HARNESS-DESIGN.md` codifies the test-harness ↔ application observability contract with naming rules, delimiter contract, must-fail-on-missing-anchor rule, self-documenting assertions, and 10 trap patterns with wrong-vs-right examples (#226, #238)

## Bug Fixes

- Test harnesses that grep for renamed/missing anchors no longer silently pass — `capture-regression.sh`, `validate-regression.sh`, and `baseline/compare-results.sh` now hard-fail on missing anchors with a diagnostic naming the expected anchor (#238)
- `validate-regression.sh` no longer aborts mid-run on a regression failure (`set -e` + `pipefail` + intentional non-zero from `diff`) (#238)
- Stray `\r…<spaces>\r` line-clear sequences no longer leak through `--disable-progress`, so section markers and other output now anchor cleanly at column 1 (#226)
- `tests/validate-index-read-back.sh` now runs in ~6 seconds (down from 5–10+ minutes) by starting each scenario from a prebuilt fixture index instead of regenerating the full index against a 290 MB log per scenario; harness auto-generates the fixture on first run via `tests/fixtures/regenerate-index-readback-fixtures.sh` (#240)
