# Release v0.14.6

## Breaking Changes

- CSV outputs (`-o`) now use consistent snake_case lowercase column names across MESSAGES and STATS, aligned with internal metric identifiers (`%log_stats` keys). MESSAGES headers `MinDuration`, `MeanDuration`, `MaxDuration`, `StdDev`, `P1`‚Äì`P99.9`, `CV`, `TotalDuration`, `TotalDurationNice`, `TotalBytes`, `TotalBytesNice`, `MeanBytes`, `Occurrences`, `Category`, `Message`, `Impact` become `min`, `mean`, `max`, `std_dev`, `p1`‚Äì`p999`, `cv`, `duration`, `duration_nice`, `bytes`, `bytes_nice`, `mean_bytes`, `occurrences`, `category`, `message`, `impact`. STATS headers `totalOccurrences`, `bytesNice`, `durationNice` become `occurrences`, `bytes_nice`, `duration_nice`. External CSV consumers must update column references. (#221)
- SRE-grade distribution-analysis columns added to `-o` CSV outputs in both MESSAGES and STATS: body percentile `p25`, precomputed `iqr`, high-volume tail `p9999`, and three shape statistics ‚Äî `skewness`, `kurtosis`, and `bimodality_coef` (Sarle's bimodality coefficient ‚Äî multimodal when > 5/9 ‚âà 0.555). Distribution shape can now be characterized from the CSV alone, enabling load-shape mode characterization, mode-evolution detection, and per-API certainty bounding without re-running ltl. The unused `z_score` column is removed from STATS (it was computed but never displayed, consumed, or documented). External CSV consumers must update column references. See `docs/usage.md` "Distribution shape" subsection for interpretation and sample-size notes. (#222)

## What's New

- `-V` now accepts an optional section argument so harnesses and humans can request specific diagnostic sections (`-V <name>`, `-V a,b`, `-V list`, `-V all`); section delimiters and names are stability-contracted (#226)
- New `tests/HARNESS-DESIGN.md` codifies the test-harness ‚Üî application observability contract with naming rules, delimiter contract, must-fail-on-missing-anchor rule, self-documenting assertions, and 10 trap patterns with wrong-vs-right examples (#226, #238)
- `-V` section `histogram` renamed to `histogram-array` to reflect the underlying architecture (exact-percentile in-memory array path) and parallel `histogram-bin-counters` (HDR-style bin-counters path); per-metric dimensions formatting is now shared between the two via a single helper so future format changes apply to both (#237)
- New `--dark-background` / `-dbg` flag forces dark heatmap gradients (inverse of `-lbg`); wins over `-lbg` regardless of CLI order. New `-V heatmap-palette` section exposes the resolved palette state (active metric, light/dark selection, source of selection, gradient arrays) and a new `tests/validate-heatmap-palette.sh` harness asserts palette correctness across `-lbg`/`-dbg` √ó `duration`/`bytes`/`count` plus both precedence orderings (#250)

## Bug Fixes

- Test harnesses that grep for renamed/missing anchors no longer silently pass ‚Äî `capture-regression.sh`, `validate-regression.sh`, and `baseline/compare-results.sh` now hard-fail on missing anchors with a diagnostic naming the expected anchor (#238)
- `validate-regression.sh` no longer aborts mid-run on a regression failure (`set -e` + `pipefail` + intentional non-zero from `diff`) (#238)
- Stray `\r‚Ä¶<spaces>\r` line-clear sequences no longer leak through `--disable-progress`, so section markers and other output now anchor cleanly at column 1 (#226)
- `tests/validate-index-read-back.sh` now runs in ~6 seconds (down from 5‚Äì10+ minutes) by starting each scenario from a prebuilt fixture index instead of regenerating the full index against a 290 MB log per scenario; harness auto-generates the fixture on first run via `tests/fixtures/regenerate-index-readback-fixtures.sh` (#240)
- `tests/validate-index-read-back.sh` assertions now self-document ‚Äî each declares the application invariant being tested, where in `ltl` it is produced, and the stability contract that makes it stable; failure output surfaces all three so failures don't require external archaeology. New `assert_command` helper added for assertions that aren't simple line greps (#239)
